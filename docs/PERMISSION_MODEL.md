## 6. 实际使用场景示例

### 场景 1: 开发人员部署到自己的服务

```
用户：alice (DevTeam_Apps 组成员)
目标：api-gw-001 服务器
  
权限链条:
  DevTeam_Apps → Developer 角色
  → 可访问标签：{env:dev, env:test, biz:apps}
  
请求访问 api-gw-001  (标签：{env:dev, biz:apps}) ✅ 允许
```

### 场景 2: SRE 跨区域操作受限制

```
用户：john (SRE_East 组成员)
目标：db-prod-primary 服务器
  
权限链条:
  SRE_East → Operator 角色
  → 可访问标签：{env:prod, region:us-east, biz:core}
  
请求访问 db-prod-wr (标签：{env:prod, region:us-west}) ❌ 拒绝 (区域不匹配)

需升级权限:
  → admin 角色 或
  → 加入 SRE_West 组
```

### 场景 3: 子业务组访问父业务组资源

```
用户：alice (payments_sre 组成员)
目标：core-auth-svc 服务器 (标签：{env:prod, biz:core})
  
权限链条:
  payments_sre → Operator + 继承父组核心权限
  → 可访问：{env:prod, biz:payments, biz:core} <-- 继承
  
请求访问 core-auth-svc (标签：{env:prod, biz:core}) ✅ 允许
  (子业务组可访问父业务组资源)
```

### 场景 4: 审计员只读访问

```
用户：sarah (Security_Auditors 组成员)
操作：查看所有设备执行历史

权限链条:
  Security_Auditors → Auditor 角色
  → 权限：[read-only, audit-read]
  
请求列表所有设备 ✅ 允许
请求对设备执行命令 ❌ 拒绝 (no execute privilege)
```

## 7. 配置文件示例 (YAML)

```yaml
# users/permissions.yaml

# AD 组映射定义
group_mappings:
  IT_Ops:
    system_role: Operator
    permissions:
      - deploy
      - config-manage

  DevTeam_Payments:
    system_role: Developer
    permissions:
      - read
      - test-deploy

  Security_Auditors:
    system_role: Auditor
    permissions:
      - read
      - audit-read

# 标签组权限矩阵
label_permissions:
  env:prod:
    groups: [ops:admin, ops:prod]
  env:dev:
    groups: [dev:admin, dev:ops]
  biz:core:
    groups: [ops:admin, ops:core]
  biz:payments:
    groups: [ops:admin, ops:payments]

# 业务组继承关系
business_hierarchy:
  biz:core:
    children:
      - biz:core-api
      - biz:core-infrastructure
      - biz:core-security
  biz:payments:
    parents:
      - biz:core  # 可以访问父组资源

# 设备标签定义
device_tags:
  - device_id: server-prod-001
    labels:
      - env:prod
      - region:us-east
      - biz:core
      - tier:critical
    
  - device_id: server-dev-001
    labels:
      - env:dev
      - biz:apps
    
    
```

## 总结

### 核心设计原则
1. **最小权限**: 用户只获得必要的权限
2. **标签隔离**: 通过标签实现资源隔离
3. **继承可控**: 子组可继承父组，但不会破坏边界
4. **审计完整**: 所有权限决策都有日志
5. **域控集成**: 从 AD 同步，减少维护成本

### 实现优先级
1. ✅ AD 认证集成
2. ✅ 基础 RBAC
3. ✅ 标签系统
4. ⏳ 继承规则
5. ⏳ 业务组映射
