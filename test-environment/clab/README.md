## 快速开始

```bash
# 安装 Containerlab
sudo ./install.sh

# 部署测试环境
sudo ./clab.sh deploy

# 测试连接
sudo ./clab.sh test

# 查看状态
sudo ./clab.sh status

# 清理环境
sudo ./clab.sh destroy
```

## 高可用特性

1. **双数据中心** - DC1 和 DC2 独立部署
2. **双 Trunk 链路** - dc1-sw1 和 dc2-sw1 之间两条 trunk 连接
3. **交换机架构** - 每个 DC 有核心交换机 + 分布交换机
4. **SSH 服务** - 每个服务器运行真实 SSH 守护进程

## 模拟服务

| 服务类型 | 模拟方式 | 连接 | 端口 |
|---------|---------|------|------|
| PhysicalHost | Docker + openssh-server | SSH 真实连接 | TCP:22 |
| NetworkDevice | Docker + net-snmpd | SNMP 真实连接 | UDP:161 |
| Container | Docker native | Docker API | - |

## 与真实环境对比

| 特性 | 真实环境 | Containerlab |
|------|---------|--------------|
| 延迟 | 真实网络延迟 | 模拟延迟 |
| 带宽 | 真实带宽 | 共享宿主机带宽 |
| 可靠性 | 硬件故障可能 | 稳定 |
| 成本 | 硬件采购 | 零成本 |
| 部署速度 | 数天 | 分钟级 |
